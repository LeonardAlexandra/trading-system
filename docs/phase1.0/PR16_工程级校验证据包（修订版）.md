# PR16：灰度演练 + 强门禁 — 工程级校验证据包（修订版）

## 1）变更摘要与边界

### 变更摘要
- **目标 1**：多重 Live 门禁（execution.live_enabled 已有；新增 allow_real_trading、live_allowlist_accounts、live_confirm_token）；**仅当 is_live_endpoint=True 时才校验**（针对 live 实盘 endpoint 的真实下单风险）；**Demo rehearsal（DEMO_LIVE_REHEARSAL）允许 OKX Demo HTTP，不触发上述门禁，不被 allow_real_trading 语义污染**。
- **目标 2**：参数精度与数量校验（qty>0、精度可配置、禁止 NaN/inf、市价单最大名义价值可选）；校验失败写 ORDER_REJECTED，不发起 HTTP。
- **目标 3**：执行模式 execution.mode（DRY_RUN | DEMO_LIVE_REHEARSAL）；演练模式下更严限频/断路器，event message 标记 rehearsal=true。
- **目标 4**：事故演练测试（OKX 5xx → 重试 → 断路器打开 → 全局拒单）；参数错误 → 本地拒绝 → 不触发 HTTP。
- **目标 5**：断路器回滚入口脚本 scripts/reset_circuit_breaker.py（受控 reset + 写 CIRCUIT_RESET_BY_OPERATOR，不含 secret）。

### 边界
- **不破坏**：PR14a live_enabled 门禁、PR15b OKX Demo 闭环、重试策略门禁、事件体系统一、默认测试离线可跑。
- **不引入**：真实实盘交易；减少或绕过既有门禁；参数校验完全交给交易所；破坏 execution_events 语义；泄露 secret/header/签名。

## 2）Live 门禁矩阵（每个门禁的触发/拒绝/审计证据）

**门禁触发条件语义（修订）**：allow_real_trading / live_allowlist_accounts / live_confirm_token **仅针对「live endpoint 真实下单风险」**；仅在 **is_live_endpoint=True** 时由执行引擎调用门禁检查。**Demo rehearsal（DEMO_LIVE_REHEARSAL）使用 OKX Demo HTTP 时 is_live_endpoint=False，不触发上述门禁，可与 real live 明确区分。**

| 门禁 | 触发条件 | 拒绝行为 | 审计证据（reason_code / event） |
|------|----------|----------|----------------------------------|
| live_enabled | PR14a 已有；策略显式 account_id + exchange_profile_id | 配置校验 fail-fast | LIVE_GATE_MISSING_ACCOUNT_ID 等 |
| allow_real_trading | **仅当 is_live_endpoint=True** 时校验；默认 false | 未开启则禁止 live 真实请求 | LIVE_GATE_ALLOW_REAL_TRADING_OFF，ORDER_REJECTED 事件 |
| live_allowlist_accounts | **仅当 is_live_endpoint=True** 时；非空时 account_id 须在列表内 | 不在列表则拒绝 | LIVE_GATE_ALLOWLIST_NOT_MATCHED |
| live_confirm_token | **仅当 is_live_endpoint=True** 时；须与环境变量 LIVE_CONFIRM_TOKEN 一致 | 不一致则拒绝 | LIVE_GATE_CONFIRM_TOKEN_MISSING |
| is_live_endpoint | PR16 仍不允许实盘 | is_live_endpoint=True 一律拒绝 | LIVE_GATE_REAL_LIVE_FORBIDDEN |

**证据**：
- 单元：`tests/unit/execution/test_live_gate.py`（dry_run 通过；**is_live_endpoint=True** 时 allow_real_trading 关/allowlist 不匹配/token 不匹配拒绝；实盘端点拒绝；is_live_endpoint=False 时 demo 路径通过）。
- 集成：`tests/integration/test_pr16_live_gates.py::test_live_endpoint_allow_real_trading_off_rejects`（模拟 is_live_endpoint=True 且 allow_real_trading=false → 拒绝，ORDER_REJECTED，无 OKX_HTTP_CREATE_ORDER）。
- **PR15b**：OKX Demo 闭环无需 allow_real_trading/live_confirm_token（demo 不触发门禁），测试全部通过。

## 4）事故演练场景与回滚说明

### 场景 1：OKX 连续 5xx → 重试 → 断路器打开 → 全局拒单
- **复现步骤**：配置 circuit_breaker_threshold=2、max_attempts=1；FakeOkx 返回 code=500；执行 2 笔决策均失败；第三笔执行时断路器已开。
- **预期 execution_events 序列**：  
  第一/二笔：CLAIMED → … → ORDER_SUBMIT_STARTED → OKX_HTTP_CREATE_ORDER / ORDER_SUBMIT_FAILED；第二笔含 CIRCUIT_OPENED。  
  第三笔：CLAIMED → … → FINAL_FAILED，reason_code=CIRCUIT_OPEN（不发起 HTTP）。
- **人工回滚/恢复**：等待 circuit_breaker_open_seconds 后断路器自动半开；或使用 **受控脚本** `scripts/reset_circuit_breaker.py <account_id>` 执行 reset 并写 CIRCUIT_RESET_BY_OPERATOR 事件（见第 10 节）；恢复 OKX 服务后重试。

### 场景 2：参数精度错误 → 本地校验拒绝 → 不触发 HTTP
- **复现步骤**：配置 order_qty_precision=4；决策 quantity=1.12345；执行 execute_one。
- **预期**：ORDER_REJECTED，reason_code=ORDER_QTY_PRECISION_EXCEEDED；无 OKX_HTTP_CREATE_ORDER。
- **人工回滚/恢复**：修正策略或配置的 qty/精度后重新发起信号；无需交易所侧操作。

**证据**：`tests/integration/test_pr16_incident_rehearsal.py::test_pr16_okx_5xx_then_circuit_open_rejects`；`tests/integration/test_pr16_param_validation.py::test_qty_precision_exceeded_rejected`。

## 10）断路器回滚入口脚本（C）— 证据

- **脚本**：`scripts/reset_circuit_breaker.py`
- **行为**：输入 `account_id`；打印当前 circuit 状态；执行 reset（归零 failures_count、清除 opened_at_utc）；写 execution_events：`CIRCUIT_RESET_BY_OPERATOR`（不含 secret）。
- **可复现步骤**：
  1. `cd trading_system`
  2. 无参：`.venv/bin/python scripts/reset_circuit_breaker.py` → 打印 Usage 并退出码 1
  3. 有参：`.venv/bin/python scripts/reset_circuit_breaker.py <account_id>` → 打印状态、执行 reset、写入事件

**示例输出片段**（无熔断记录时）：

```
account_id=test-acc-1
current state: no row (no failures recorded)
reset done.
event written: CIRCUIT_RESET_BY_OPERATOR decision_id=operator-reset-test-acc-1-20260202151710
```

**有熔断记录时**：`current state: failures_count=N, opened_at_utc=...`，随后 reset done 与 event written。
