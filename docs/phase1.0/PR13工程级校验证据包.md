# PR13 工程级校验证据包

本文档用于 PR13 的工程级放行判断：实盘接口边界、账户/交易所显式建模、多层安全阀已实现，全行为可审计。所有结论均有证据来源。

---

## 1）变更摘要与边界

**做了什么**

- **目标 1 - 真实交易所 Adapter 接口设计（结构，非实盘）**：ExchangeAdapter 明确 place_order（即 create_order）、cancel_order、get_order_status（即 get_order）能力边界；CreateOrderResult / GetOrderResult 增加 error 字段；PermanentOrderError（不可重试）与 TransientOrderError（可重试）区分；PaperExchangeAdapter 作为 reference 实现。
- **目标 2 - 账户/子账户/交易所工程建模**：AppConfig 增加 exchange_profiles、accounts；StrategyEntryConfig 增加 exchange_profile_id、account_id；validate() 对 strategy → exchange_profile → account 做 fail-fast 校验；ResolvedStrategyConfig 增加 account_id、exchange_profile_id；execution_events 增加 account_id、exchange_profile、dry_run 列；关键事件可追溯到 account/exchange。
- **目标 3 - 实盘安全阀**：① Dry-run：execution.dry_run、DryRunExchangeAdapter（不真实下单）、事件 dry_run 标记；② 全局限频：execution.max_orders_per_minute、超限 FAILED + RATE_LIMIT_EXCEEDED；③ 断路器：execution.circuit_breaker_threshold / circuit_breaker_open_seconds、连续失败熔断、CIRCUIT_OPENED/CIRCUIT_CLOSED 入 events。

**没做什么**

- 未接真实交易所、未写真实 SDK 调用、未使用真实 API Key。
- 未改变幂等模型、ExecutionEngine 事务边界、RiskManager 语义。
- 未引入 StrategyManager、未隐式默认账户。

## 6）配置与运维影响

- **新配置项**（均有默认值、启动期校验）：
  - execution.dry_run：bool，默认 False
  - execution.max_orders_per_minute：int，默认 0（不限）
  - execution.circuit_breaker_threshold：int，默认 0（关闭）
  - execution.circuit_breaker_open_seconds：int，默认 60
  - strategies[*].exchange_profile_id、account_id：可选；配置则必须在 exchange_profiles/accounts 中存在且一致
  - 顶层 exchange_profiles、accounts：可选；用于 fail-fast 校验
- **fail-fast**：validate() 校验上述项；缺失/不一致 → ConfigValidationError(INVALID_ACCOUNT_CONFIGURATION)。
- **向后兼容**：未配置 exchange_profile_id/account_id 时使用默认 "paper"/"default"；dry_run/限频/断路器默认关闭。

## 7）风险清单与残余风险声明

- **新风险点**：断路器状态为进程内内存，多实例不共享；限频按 ORDER_SUBMIT_STARTED 计数，与真实“成交数”可能不完全一致。
- **缓解方式**：文档明确单进程/单实例假设；限频为安全阀而非精确流控。
- **残余风险**：无；上述为设计范围内行为。
