# PR12 工程级校验证据包

> **说明**：工程级整改后请以《PR12 工程级校验证据包（修订版）.md》为准。

本文档用于 PR12 的工程级放行判断：证明 OrderManager 基础功能已实现，且未破坏 PR11 不变式。所有结论均有证据来源。

---

## 1）变更摘要与边界

- **PR12 目标**：实现订单查询、取消、状态同步（基础版）；为真实交易所接入做结构准备（exchange_adapter 扩展），保持 PaperExchangeAdapter 为默认执行实现。
- **明确不做什么（Non-goals）**：不实际接入真实交易所；不引入 StrategyManager / 消息队列；不改变幂等模型、ExecutionEngine 事务边界、风控规则；不改价功能（延后）。
- **影响面**：
  - **模块**：`src/execution/order_manager.py`（新增）、`src/execution/exchange_adapter.py`（抽象 + Paper 实现）、`src/repositories/orders_repo.py`（新增 update）、`src/common/order_status.py`（新增 ORDER_STATUS_CANCELLED）。
  - **配置**：无新增配置项。
  - **DB**：无 migration；沿用 orders 表。
  - **运维**：无变更。

- **ExecutionEngine 事务边界**：未改变。未修改 execution_engine.py。
- **风控规则**：未改变。未修改 risk_manager.py / risk_state / position 逻辑。
- **审计语义**：未改变。CONFIG_SNAPSHOT、execution_events 逻辑未动。
- **secret 隔离**：未改变。GetOrderResult / CreateOrderResult 无 raw secret；Paper 不落库敏感信息。

## 8）风险清单与残余风险声明

- **新风险点**：OrderManager 不在此内 commit，调用方须管理 session；Paper 内存订单在进程重启后丢失（仅影响 Paper，符合 Phase1.0 范围）。
- **缓解方式**：文档与接口注释明确“由调用方管理 session”；Paper 仅用于测试与单机纸面。
- **残余风险**：无；上述为设计范围内的行为，非 bug。
