# PR12 工程级校验证据包（修订版）

本文档用于 PR12 工程级整改后的放行判断：OrderManager 行为进入 execution_events 审计链，与 ExecutionEngine 的职责边界已代码化并覆盖测试。所有结论均有证据来源。

---

## 1）变更摘要与边界

- **PR12 目标**：实现订单查询、取消、状态同步（基础版）；**工程级整改**：OrderManager 关键行为（cancel / sync）写入 execution_events，可审计、可回放、可解释；明确 OrderManager 与 ExecutionEngine 的职责边界。
- **明确不做什么（Non-goals）**：不接入真实交易所；不引入新配置、不重构 ExecutionEngine、不改变幂等模型；不绕过审计（必须走 execution_events）。
- **影响面**：
  - **模块**：`src/execution/order_manager.py`（注入 event_repo，cancel/sync 追加事件）、`src/common/event_types.py`（ORDER_CANCELLED / ORDER_SYNCED）、`tests/execution/test_order_manager.py`（断言事件内容）、`tests/integration/test_order_manager_audit.py`（新增，Engine → OrderManager → events 顺序与订单状态）。
  - **配置**：无新增。
  - **DB**：无 migration；沿用 execution_events 表（无新列，message 承载 order_id/strategy_id/old_status/new_status）。
  - **运维**：无变更。

---

## 3）回归不变式声明

- **幂等语义**：未改变。ExecutionEngine 仍以 decision_id 为幂等主键；OrderManager 不参与 execute_one 流程。
- **ExecutionEngine 事务边界**：未改变。未修改 execution_engine.py。
- **风控规则**：未改变。
- **审计语义**：**增强**。execution_events 新增 ORDER_CANCELLED / ORDER_SYNCED；事件仍按 decision_id 查询、created_at 递增，无回写。
- **secret 隔离**：未改变。事件 message 仅人类可读、不含敏感信息。

## 8）风险清单与残余风险声明

- **新风险点**：OrderManager 依赖 event_repo，调用方须传入同一 session 的 event_repo 并在外部 commit。
- **缓解方式**：模块注释与证据包明确“不在此内 commit”；集成测试覆盖同一 session 下 order_repo + event_repo 行为。
- **残余风险**：无。
