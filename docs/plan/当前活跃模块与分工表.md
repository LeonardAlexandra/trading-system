## 一、角色约定（固定）

| 场景 | 开发者（开发/修改） | 审计者（系统级审计） |
|------|---------------------|----------------------|
| **各 Phase 内单模块** | Cursor | OpenClaw |
| **Phase 完成后整体测试与验收** | Cursor（执行测试、修 bug、补充证据） | OpenClaw（系统级审计） |
| **全系统测试与验收**（所有 Phase 完成后） | Cursor（执行全系统测试、修 bug、补充证据） | OpenClaw（系统级审计） |

### 门禁（强制）

- **当前模块经 OpenClaw 通过后，Cursor 方可进入下一模块。**  
  Cursor 可**主动问询** OpenClaw 当前模块是否通过，或**等待** OpenClaw 回传/更新审计结论；在未收到 OpenClaw 的「通过」确认前，Cursor **不得**开始下一模块的开发、测试或修改。  
  OpenClaw 审计通过后，应更新分工表或明确回传通过结论，以便 Cursor 进入下一模块。

## 八、更新本表的责任

- **单模块完成后**：**开发者（Cursor）** 在交付证据包并自检通过后，将本表「二、当前状态」中的下一待做模块更新为**紧接着的下一项**，将「上一模块证据包路径」更新为刚完成的证据包路径；若该模块需**审计者（OpenClaw）** 审查，则状态可先标为「待审计」，审计通过后再标为「已完成」并更新当前状态。
- **Phase 整体测试与验收 / 全系统测试与验收**：由 **Cursor** 在执行完测试与修复后更新对应阶段状态；**OpenClaw** 完成系统级审计后，在相应验收报告或本表备注中注明审计结论。
